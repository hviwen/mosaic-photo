# Augment 团队开发规范

## 目标：保持回答和代码风格一致、简洁、安全、可复用，遵循团队 UI 体系。

## 主要语言与呈现

1. 会话与普通说明：**始终使用简体中文回答（中文）**。
2. 代码中的注释、文案（strings、README 文段等）：**默认使用中文并保留编码为 UTF-8**，除非任务明确要求"注释/文案使用英文"。

## 代码质量与实现要求

1. 不要过度设计，保证代码**简洁易懂**，以简单实用为主。
2. 写代码时尽量降低圈复杂度（Cyclomatic Complexity），Favor clarity over cleverness。
3. 注重复用 — 抽取公共函数/组件，避免重复实现。
4. 注意模块划分与边界，优先使用常见设计模式（Factory, Strategy, Adapter, Observer 等）解决重复结构问题。
5. 改动最小化原则：在实现或修复时尽量避免改动无关模块代码；如果必须大范围改动，请在回复里明确列出改动影响点与回滚计划。

## 交付格式与示例

1. 若返回代码，请提供：功能说明（中文），注意事项（中文），以及可运行的代码块。
2. 若进行架构或接口变动，给出迁移/回滚建议与影响范围估计。

## 项目特定规范

### Vue 3 + TypeScript 开发规范

- 使用 Composition API，避免 Options API
- 组件命名使用 PascalCase
- 合理使用 TypeScript 类型注解，避免 any
- 状态管理使用 Pinia

### 样式规范

- 优先使用 Tailwind CSS 类名
- 复杂样式使用 SCSS 模块化
- 保持响应式设计考量

### 文件组织

- 按功能模块分组，不按文件类型
- 工具函数放在 utils/ 目录
- 类型定义统一在 types/ 目录
