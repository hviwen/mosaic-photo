# Claude 代码助手指令

## 核心原则

遵循团队开发规范，保持代码风格一致、简洁、安全、可复用。

## 语言使用规范

- **回复语言**：始终使用简体中文进行交流
- **代码注释**：默认使用中文注释，UTF-8 编码
- **变量命名**：使用英文，遵循 camelCase/PascalCase 规范
- **文档撰写**：中文说明 + 英文技术术语

## 代码实现要求

### 质量标准

1. **简洁性**：避免过度设计，优先选择简单直接的实现方式
2. **可读性**：代码逻辑清晰，命名语义化，适当添加注释
3. **复用性**：提取公共逻辑为函数/组件，避免重复代码
4. **可维护性**：模块边界清晰，职责单一

### 技术实现

- 降低圈复杂度，避免深层嵌套
- 合理使用设计模式解决架构问题
- 遵循 SOLID 原则
- 错误处理要完善，边界情况要考虑

### 改动控制

- **最小化原则**：只修改必要的代码
- **影响评估**：大范围改动需要说明影响范围
- **回滚计划**：提供变更的回滚方案

### 文档交付

1. **功能说明**（中文）
2. **使用示例**（带注释）
3. **注意事项**（中文）
4. **API 接口说明**

### 架构变更

- 变更影响范围评估
- 迁移步骤说明
- 回滚计划制定
- 测试验证方案
