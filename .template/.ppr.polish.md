## 背景
这是一个基于 Vue 3 + TypeScript + Vite 的照片拼贴应用。当前已在 `src/utils/smartCrop.ts` 中实现了基于人脸检测的智能裁剪功能，包括：
- 默认使用浏览器原生 `FaceDetector` API（Shape Detection API）
- 支持通过 `setFaceDetectionProvider()` 注入第三方人脸检测库
- 已实现"显著性对象检测"作为兜底方案
- 在 `src/composables/useLayout.ts` 的 `fillArrangePhotos()` 中调用智能裁剪

## 当前问题
根据 `.template/.ppr.next.md` 中的描述，竖向人像图片在被裁剪为横向比例时，人脸经常被截断或显示不完整。

## 优化目标
评估并推荐更成熟的人脸/对象检测库，用于替代或增强当前的检测能力，具体要求：

### 1. 技术选型建议
请评估以下方案的优劣，并给出推荐：
- **MediaPipe Tasks Vision (FaceDetector)** - Google 官方，跨浏览器兼容性好
- **face-api.js** - 基于 TensorFlow.js，生态成熟
- **TensorFlow.js + COCO-SSD** - 可检测多种对象（人、动物等）
- 其他更适合的方案

### 2. 评估维度
- **检测精度**：人脸、动物脸部的检测准确率
- **性能开销**：模型大小、初始化时间、检测速度
- **浏览器兼容性**：是否需要 WebAssembly/WebGL 支持
- **集成难度**：与现有 `FaceDetectionProvider` 接口的适配成本
- **bundle 体积影响**：是否支持懒加载/按需加载

### 3. 集成方式要求
- 必须通过 `setFaceDetectionProvider()` 注入，避免强依赖
- 需要支持懒加载（避免影响首屏加载速度）
- 模型文件应放在 `public/` 目录并支持 CDN 缓存
- 需要配合现有的 `deadlineMs`（默认 450ms）超时机制

### 4. 实现示例
请提供推荐方案的集成代码示例（伪代码即可），包括：
- 如何在 `src/main.ts` 或独立插件文件中注入 provider
- 如何处理模型文件的懒加载
- 如何将检测结果映射为 `FaceDetectionProviderResult[]` 格式

### 5. 动物检测支持
如果推荐的方案支持动物脸部/对象检测，请说明：
- 如何区分人脸和动物脸
- 是否需要修改 `SmartDetection` 类型定义（当前只有 `'face' | 'object'`）
- 如何在 `calculateSmartCrop()` 中统一处理人脸和动物脸
